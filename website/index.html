01 / 核心能力

从一个URL，
到生产级流量。

先做干净的基准测试，再把真实请求序列带进来。每一步都可复现、可解释、可验证。

01

回放你的日志

将 Nginx common/combined 日志转换成可重复工作负载。顺序、shuffle、seeded random 和时间戳节奏。

--access-log production.log
--replay-timestamps --replay-speed 2
02

塑造流量阶段

用 stages 定义基线、尖峰和恢复速率，或输出稳定 JSON 结果接入 CI 和 GUI。

--replay-stages 10s:100,5s:1000
03

看见完整结果

HDR 延迟分布、状态码、method 摘要、URI 热点、JSON schema 和分类 socket 错误。

04

面对故障仍可继续

时长模式恢复 reset、timeout、拒绝连接和服务重启，同时保留错误统计。

connect · read · write · timeout

03 / 开始使用

一条命令，
得到第一个信号。

通过 Homebrew 或 crates.io 安装最新版本，也可以从 GitHub Releases 获取发布包。

Homebrew ↗crates.io ↗docs.rs ↗GitHub Releases ↗
# macOS / Linux（Homebrew）
brew install wenhaozhao/rload/rload

# Rust 用户
cargo install rload

# JSON 结果
rload -d30s --output-format json https://api.example.com

# 回放生产流量
rload --access-log ./access.log https://staging.example.com
