01 / 核心能力

从一个URL，
到生产级流量。

先做干净的基准测试，再把真实请求序列带进来。每一步都可复现、可解释、可验证。

01

回放你的日志

将Nginx common/combined日志转换成可重复工作负载。顺序、shuffle或带seed的随机顺序。

--access-log production.log
--replay-order shuffle --seed 42
02

塑造每个请求

curl风格method、header、UTF-8 body、二进制payload和JSONL请求文件。

-X POST -H 'Content-Type: application/json'
03

看见完整结果

HDR延迟分布、状态码、method摘要、URI热点和分类socket错误。

04

面对故障仍可继续

时长模式恢复reset、timeout、拒绝连接和服务重启，同时保留错误统计。

connect · read · write · timeout

03 / 开始使用

一条命令，
得到第一个信号。

从crates.io安装最新版本，或从GitHub Releases获取发布包。

crates.io ↗docs.rs ↗GitHub Releases ↗
# 安装
cargo install rload

# 压测服务
rload -t2 -c10 -d30s https://api.example.com

# 回放生产流量
rload --access-log ./access.log https://staging.example.com
