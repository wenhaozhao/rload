01 / 核心能力

从单点压测，
进化到生产级真实流量回放。

传统的循环刷单 URL 无法暴露真实瓶颈。rload 让每一笔请求、每一个入参和调用顺序都源于真实环境，且完全幂等可复现。

01 / HIGH FIDELITY

Nginx 访问日志高保真回放

原生流式解析 Common/Combined Nginx 日志，无需编写任何 Lua 脚本。支持顺序、洗牌（Shuffle）和随机三种回放模式。

--access-log access.log --replay-order shuffle --seed 42
02 / CUSTOM PAYLOADS

结构化 JSONL 请求回放

支持回放包含自定义 Method、Headers、和 UTF-8 Body 的结构化 JSON 列表。通过高精度时间轮优雅模拟真实用户请求间隔。

--request-file queries.jsonl --replay-timestamps
03 / PERFORMANCE PARITY

与 wrk 性能及指标绝对对齐

基于 Bare-metal 裸 mio 实现多工作线程绑核模型。实测常驻内存仅 3.5MB，吞吐量 RPS 相比 wrk 误差低于 1%，零调度器额外噪声。

rload (3.55MB) wrk (3.47MB) k6 (>100MB)
04 / REAL SRE WORKFLOWS

防协同遗漏与流量塑造

内置精准的发送调速步调，真实还原服务端卡顿时客户端的排队延迟。支持通过 stages 定义阶梯式、突发式等高阶负载波形。

--replay-stages 10s:100,5s:1000

04 / 经典实战指令

常见工程压测场景，
复制即用。

🔥 场景 A：静态极速压测（对齐 wrk）

使用 4 个工作线程、200 个连接，对目标接口压测 30 秒，并持续打印延迟统计：

rload -t 4 -c 200 -d 30s https://api.yourdomain.com

📂 场景 B：高保真 Nginx 日志随机回放

读取真实的 Nginx 访问日志，使用特定随机种子（确保结果可幂等重现）打乱顺序进行回放：

rload -c 100 -d 60s --access-log ./access.log --replay-order shuffle --seed 12345 http://staging.internal

⏳ 场景 C：按日志真实时间戳速度回放

按照 Nginx 日志中记录的相对时间差高保真还原流量。使用双倍速进行压力测试：

rload -c 50 --access-log ./access.log --replay-timestamps --replay-speed 2.0 http://staging.internal

📝 场景 D：JSONL 多形态动态请求回放

顺序循环回放自定义导出的 JSON 结构化请求文件（支持自定义 Header、Method 和 Request Body）：

rload -c 100 -d 5m --request-file ./requests.jsonl http://staging.internal
